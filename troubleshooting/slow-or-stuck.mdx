---
title: "Slow or Stuck"
description: "Tell a normal MindMap pause from a real hang, handle long tool calls and approval waits, and learn how to stop a run."
---

A MindMap pause usually means it's working, not broken. The trick is telling a normal pause apart from a genuine hang. Work down this page in order.

## It paused on a tool call

When MindMap searches the web, reads a calendar, or opens a connector, it shows a tool-call chip and runs that step in the background. This is the most common "it stopped" moment, and it's normal. Reads finish on their own and the answer follows.

Give a tool call 10 to 30 seconds. A web search or a slow connector can run longer than a plain reply, so a single chip sitting for a bit is fine.

[What tool calls are](/understanding/tool-calls)

## It's waiting on your approval

MindMap queues every write (sending mail, creating a record, posting to a connected app) instead of running it. If it looks done but nothing happened, check the approvals panel: there may be a proposed action waiting for your **Approve** or **Reject**.

Nothing leaves your workspace until you approve it, so a stalled write is often just a pending approval you haven't acted on yet.

## How to stop a run

If MindMap is mid-run and you want it to halt:

- Use the **Stop** control on the active message to cancel the current run.
- If a write is queued, **Reject** it in the approvals panel so it never runs.

Stopping a run cancels the in-progress work. It doesn't undo anything already approved and carried out.

## It's been over a minute with nothing

If it's clearly past a normal tool call and there's no pending approval:

<Steps>
  <Step title="Refresh and resend">
    Reload the page and send your message again. A long task can occasionally drop its stream; resending picks it back up.
  </Step>
  <Step title="Check your connection">
    MindMap runs in the cloud and streams replies to your browser. A slow or dropped connection looks like a slow assistant.
  </Step>
  <Step title="Break up a big request">
    A single sprawling request takes longer and stalls more easily. Split it into smaller steps and send the first one.
  </Step>
</Steps>

## It stopped mid-reply

If a response cut off partway, type "continue" or "keep going" and MindMap resumes from where it left off.

## Nothing responds at all

If the chat is fully frozen (typing does nothing, no chip, no reply):

1. Hard refresh the page (Ctrl+Shift+R on Windows, Cmd+Shift+R on Mac).
2. Clear your browser cache if the hard refresh doesn't help.
3. Try a different browser.

<Note>
If replies feel slow across the board, an admin can switch the model in Admin > Agent. The model switcher changes which model MindMap runs per surface, which can trade speed for depth. That's an admin setting, not something you change per message.
</Note>

If none of this clears it, use the in-app feedback button or tell your admin what you saw (a stuck chip, a pending approval, or a fully frozen chat) so they can check the tenant.
