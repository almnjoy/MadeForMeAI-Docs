the infra-health agent files (direct)


One bundle for the operator's own monitoring stack. It combines two things that used to be
separate features:

- **Infra metrics**: PromQL queries against Prometheus, LogQL queries against Loki, and saved
  Grafana dashboards rendered as tiles.
- **Infra agents**: the infra-health snapshots and helpdesk cases the T1/T2 monitoring agents
  produce (node status, container checks, recent warnings, open incidents).

This is a **hidden-tier** plugin: it does not appear in the marketplace for customers. It is
operator/custom, enabled per box out of band. The Infra page itself is built per customer.

## HOW TO SETUP

This one is not self-serve from the marketplace. To turn it on for a box:

1. Set the feature flag on: `features.infra: true` in that box's tenant config (rocker and the
   devzone MindMap dev box have this in `tenant.demo.json`), or flip it in the settings table.
2. Point it at the stack via env / tenant config: `PROM_URL`, `LOKI_URL`, the Grafana base
   (`infra.grafanaBase`), and the saved dashboards list (`infra.dashboards`).
3. Set the OPS_CENTER node cards via `infra.nodes`: a list of `{ key, label, selector }` where
   `selector` is a brace-less PromQL label set, e.g.
   `instance="controlnode",job="integrations/unix"`. The cards run CPU/MEM/DISK/LOAD against
   each selector. If `infra.nodes` is empty the code falls back to neutral placeholders.
4. Each `infra.dashboards` entry may carry a `params` string of Grafana template vars appended
   to the embed URL, e.g. `var-instance=10.20.10.201` (Proxmox) or
   `var-ds_prometheus=<uid>&var-job=integrations%2Funix` (Node Exporter Full). The embed drops
   full `kiosk` so the dashboard's variable dropdowns stay usable in the frame.
5. For the agents side, mount the infra-health directory (the agents' `latest.md` + hourly
   snapshots + `helpdesk/cases/`) at `/infra-health` in the api container.

**Embedding Grafana cross-site** (when the app and Grafana are on different domains): Grafana
needs `allow_embedding=true` AND either anonymous Viewer access or a same-site session, and the
app's own CSP must permit it (`frame-src 'self' https:` in `apps/web/nginx.conf`). A missing
`frame-src` falls back to `default-src 'self'` and the browser blocks the frame with
"This content is blocked".

On your own box (`MARKETPLACE_OWNER=true`), the Infra card shows in the marketplace under
**Hidden · owner only** when you load it; on a customer box it never appears.

## WHAT USER NEEDS TO GET

- **A Prometheus URL** scraping node_exporter (`PROM_URL`).
- **A Loki URL** for logs (`LOKI_URL`).
- **A Grafana base URL** plus the dashboards you want to render (uid + slug + key, optional
  `params`), in the tenant config `infra.dashboards`.
- **The node list** for the OPS_CENTER cards (`infra.nodes`): label + PromQL selector per node.
- For the agent snapshots/cases: the **infra-health files** mounted into the container (synced
  from wherever the agents write them if the box is not the writer, e.g. the devzone cron).

## HOW IT WORKS

- `query_metric` runs an instant PromQL query. Only **node_exporter host metrics** are scraped
  (CPU, memory, disk, load, network, up). There are no container/cAdvisor or application
  metrics, so the agent is told to say so rather than invent a metric name.
- `query_logs` runs a LogQL selector against Loki (labels: host, container, namespace,
  service_name, job), newest lines first.
- `show_dashboard` renders one of the saved Grafana dashboards as an embedded tile.
- `infra_health` reads the latest agent snapshot; `infra_cases` lists recent helpdesk cases;
  `read_infra_file` opens a specific snapshot or case file by name.
- Everything here is **read-only**. The Infra page and the `/api/infra/*` endpoints stay in the
  app; this plugin is the agent-tools side.

## WHAT CAN BE CALLED VIA AI

- "what is CPU on node-1" / "memory free on node-2": PromQL metric
- "any errors in the app logs in the last hour": Loki logs
- "show the node overview dashboard": Grafana tile
- "how is the infra" / "what do the agents report": latest health snapshot
- "any open cases" / "recent incidents": agent helpdesk cases

## LIMITATIONS

- **node_exporter metrics only.** No container or application metrics. Asking for one that does
  not exist gets a plain "not available", not a guess.
- **Dashboards must be in the tenant config** (uid + slug + key). An unknown key returns the
  list of valid ones.
- **The agents side needs the mount.** Without the infra-health directory mounted, the
  health/cases tools have nothing to read.
- **Hidden / not self-serve.** It is enabled per box by you, not from the customer marketplace.
- **The Infra page is custom per customer.** The metric/log tools are generic; the page is not.

## FAQ

**Why is Infra hidden?** It is operator/custom for now. The page is built per customer and the
tools point at your own stack, so it is not a turn-key marketplace add-on yet.

**How do I enable it for a specific user?** Set `features.infra: true` and the PROM/LOKI/Grafana
config on that user's box, plus the infra-health mount for the agents side. They still will not
see a marketplace card unless their box is the owner console.

**Does it change anything?** No. Every tool is read-only: metrics, logs, dashboards, snapshots,
and case files.

**Can I see the Infra card on my own box?** Yes, on the owner box it appears under "Hidden ·
owner only" in the marketplace.
