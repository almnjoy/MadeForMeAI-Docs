Object storage for files your agents generate: proposals, exports, images, PDFs. Browse, preview,
download, upload, delete. S3-compatible, so it runs on MinIO now and AWS S3 later with no code
change.

## HOW TO SETUP

1. Enable the **Assets** app in the marketplace.
2. Add the secrets: `S3_ENDPOINT` (e.g. `https://s3.madeformeai.com`), `S3_ACCESS_KEY`,
   `S3_SECRET_KEY`, `S3_BUCKET`.
3. The Assets page + nav appear, and the agent gets `save_asset` / `list_assets`.

## WHAT USER NEEDS TO GET

An S3-compatible endpoint + access key/secret + a bucket. On the MadeForMeAI cloud this is
provisioned per box on worker2 (a bucket + a bucket-scoped user per tenant).

## HOW IT WORKS

- The Assets page lists objects with size, source (agent or upload), and date. Click a file to
  preview (images render inline) or download.
- Downloads use a short-lived presigned URL straight from storage, so big files do not stream
  through the app.
- Upload via the page (pick a file) or have the agent `save_asset` a generated document. Each file
  carries provenance metadata (which agent/skill made it).

## WHAT CAN BE CALLED VIA AI

- "save this as an asset" / "export this proposal to a file" -> `save_asset`
- "what files do I have in Assets" -> `list_assets`

## LIMITATIONS

- Web upload is a single file up to 25 MB (no multipart or folder upload yet).
- Top-level keys; no folder browser yet.
- The agent saves text/markdown content; binary outputs (images, PDFs) come from whatever feature
  generates them, then land here.

## FAQ

**Where are the files?** In your S3 bucket (MadeForMeAI cloud: worker2 MinIO, one bucket per box).

**Is the bucket public?** No. It is private; downloads use short-lived presigned URLs, and each
tenant's key can see only its own bucket.

**Can I see the bucket directly?** The MinIO console is internal by default. Ask to expose an
`s3-console.*` route if you want the admin UI.
