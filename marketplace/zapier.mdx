Zapier server)


Let the agent use your Zapier-connected apps. MindMap connects to your Zapier MCP server and
discovers the actions you have enabled there, then exposes them to the agent. Read actions (find
an email, look something up) run immediately; write actions (send an email, create a record) are
approval-gated, so the agent queues them and they only fire after you approve. Whatever actions
you add on the Zapier side show up here automatically, so you can wire up many without any
per-action setup in MindMap.

## HOW TO SETUP

1. In Zapier, open your **MCP Server** (mcp.zapier.com). On the **Connect** tab, set the transport
   to **Streamable HTTP** and copy the **server URL** (the token is baked into the URL).
2. On the **Apps** tab of that server, click **+ Add tool**, search the app (e.g. Gmail), pick the
   action (e.g. Send Email), connect the account, and save. Add as many as you want.
3. In MindMap, go to **Admin** (marketplace) and open the **Zapier** card.
4. Paste the **Zapier MCP URL** into its write-only box and **Save**.
5. Flip **Enable** on.

It applies on your next message. New actions you add on the Apps tab appear in MindMap within
about a minute (a short discovery cache).

## WHAT USER NEEDS TO GET

- **A Zapier MCP server URL** (Streamable HTTP) from the Connect tab. The URL carries its own
  token, so it is the credential. You can rotate it any time with Zapier's Rotate token button.
- **The actions you want, added on the Apps tab** of that server. This is the important part:
  Zapier MCP exposes the **actions you add there**, not your Zaps. Building a Zap does not make it
  show up here. Add the app action (Gmail Send Email, etc.) on the Apps tab.

## HOW IT WORKS

- MindMap is an **MCP client** to your Zapier MCP server. It reads the URL via the marketplace
  key store (live, no restart) and handshakes the server to discover your enabled actions.
- Zapier MCP uses a **dynamic execution** model, not one tool per action. The agent calls
  `list_enabled_zapier_actions` to see what you enabled, `execute_zapier_read_action` to run a
  read, and `execute_zapier_write_action` to run a write.
- MindMap runs the **reads immediately** and **approval-gates the writes**: a write queues a
  proposal, and only fires through your approval (the same trust model as every other write in the
  app). The agent never sends or creates anything on its own.
- Zapier's server-management and skill tools are hidden from the agent, so it only sees the parts
  that act in your connected apps.
- Credentials for the underlying apps (your Gmail, etc.) stay on Zapier's side. MindMap only
  holds the MCP URL.

## WHAT CAN BE CALLED VIA AI

Whatever actions you enabled on the Apps tab. For example, with Gmail actions added:

- "find the latest email from <person>" (runs immediately, read)
- "send an email via Zapier to <addr> with subject X and body Y" (queues an approval, sends on
  approve, write)
- "create a draft reply to <thread>" (queued for approval)

The agent will list your enabled actions first, then act. Reads return data; writes wait for you.

## LIMITATIONS

- **Only the actions you add on the Apps tab** are available. Your Zaps are not exposed through MCP.
- **Writes are approval-gated** (by design to start). A per-action "allow always" toggle can be
  added later if you want certain low-risk writes to skip approval.
- **About a minute of lag** when you add a new action (the discovery cache refreshes on a TTL).
- **One MCP URL per box.** The URL is the credential; rotate it on the Zapier side if it leaks.
- This is the **MCP** path. For hand-built flows you would rather own end to end, n8n is the other
  lane (the Automations feature).

## FAQ

**My Zap does not show up.** Right, Zapier MCP exposes the **actions you add on the Apps tab**, not
Zaps. Add the app action there (e.g. Gmail Send Email) and it appears here within ~60s.

**How do I add more actions?** On your Zapier MCP server's Apps tab: + Add tool, pick the app +
action, connect, save. No MindMap change needed.

**Can the agent send things on its own?** No. Write actions are queued for your approval and only
fire after you approve. Reads run on their own.

**Where does my token live?** It is part of the MCP URL, saved write-only to your pod, never shown
back, never in the repo. Rotate it on Zapier with Rotate token if needed.

**Zapier or n8n?** Both work. Zapier here is the MCP path (fast to wire many app actions); n8n is
for flows you build and host yourself (the Automations feature).
