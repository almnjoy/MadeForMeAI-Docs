# Review Requests

After a paid job, shoot, or milestone, draft an approval-gated ask for a Google or Yelp review with
your review link. Tracks who was asked. The owned-reputation flywheel every persona needs.

## HOW TO SETUP
Marketplace -> Review Requests. Optionally set `REVIEW_URL` (your Google/Yelp review link) so asks
prefill it. No key required.

## WHAT USER NEEDS TO GET
Your public review link (optional but recommended).

## HOW IT WORKS
`request_review` drafts the message and queues an approval. On approve, the request is marked sent
and returns the message + link to send through your channel (Telegram, n8n, or Zapier). Stored in the
`review_requests` table.

## WHAT CAN BE CALLED VIA AI
- "Ask Acme for a review, text 303..." -> request_review (approval-gated)
- "Who have I asked for reviews?" -> list_review_requests

## LIMITATIONS
- The plugin drafts and tracks; sending the SMS/email routes through your channel (no direct SMS
  sender wired unless you use Telegram/n8n/Zapier).
- No automatic "after paid invoice" trigger yet; pair with the AR watcher or a saved play + n8n
  schedule to automate the timing.

## FAQ
Q: Can it auto-send after a job? A: Draft + queue yes; automatic timing via an n8n schedule that
asks the agent to run the review play.
