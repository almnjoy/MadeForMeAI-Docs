An Obsidian-style force graph of your notes. Each note is a node, each `[[wikilink]]` inside a
note is an edge. Click a node to read or edit its markdown. It is the in-app view of your notes
vault, the most on-brand surface for a product named MindMap.

## HOW TO SETUP

1. Enable the **Memory View** app in the marketplace (Admin -> Apps). The Memory page + left-nav
   item appear.
2. It reads notes from the **Notes** tool. For real two-way notes, also connect the **WebDAV
   Vault** app (otherwise notes fall back to a local database table). See `WebDAV-Vault.md`.

## WHAT USER NEEDS TO GET

Nothing external. Notes come from the vault (or the local table). To make the graph interesting,
write a few notes and link them with `[[Title]]`.

## HOW IT WORKS

- Nodes = notes; a node with more links is bigger. Green = linked, gray = orphan, dashed = an
  unresolved `[[link]]` target (click it to create that note).
- Controls panel (the sliders icon): Groups legend, Display (node size, labels, show orphans),
  Forces (repel, link distance), Fit to view.
- The search box dims non-matching nodes. Click a node to open the markdown editor (edit / save /
  delete). New note button top-left.
- Rendered with react-force-graph (the same d3-force-on-canvas engine Obsidian's graph uses).

## WHAT CAN BE CALLED VIA AI

The Memory page is a UI. The canvas agent works the same notes through the Notes tool: "add a
note for X", "what links to Y", "append this to my Z note".

## LIMITATIONS

- Top-level notes only for now (Obsidian subfolders are not recursed yet).
- The graph fetches each note body to draw edges, so a very large vault loads more slowly.
- It is a page app: it does not add an agent tool of its own (that is the Notes tool).

## FAQ

**Where do the notes live?** In the WebDAV Vault when connected (two-way with Obsidian), otherwise
a local DB table.

**Why is a node dashed?** It is a `[[link]]` target with no note yet. Click it to create the note.
