Run an agent instruction on a schedule: once, daily, weekly, or every N days. The job runs your prompt through the agent and routes any writes through your Trust Rules: a write auto-runs only where a rule allows it, otherwise it queues for approval. Owner-only and off by default.

## What you need

Nothing to supply, it's built in. (Owner box only.)

## Setup

<Steps>
  <Step title="Enable the app">
    Admin, then Apps, then Scheduled Jobs, then enable. This app is hidden and only reveals on the owner box.
  </Step>
  <Step title="Create a job">
    A Jobs item appears in the sidebar. Open it and click New job. Give it a title, the agent instruction (e.g. "Email a summary of today's pipeline to me@x.com"), and a schedule (once / daily / weekly / interval, with an optional max-runs or end date).
  </Step>
  <Step title="Add a Trust Rule for unattended writes">
    To let scheduled writes go out unattended, add a matching Trust Rule (Admin, then Trust), for example approve one send with "Always allow this", or set a campaign rule with a use-count. Without a rule, each scheduled write queues for your approval.
  </Step>
</Steps>

## How it works

- A single in-process loop checks for due jobs each minute. It runs the job's prompt, dispatches proposals through Trust Rules, records the result, and advances `next_run_at` (or completes a one-time or capped job).
- Schedules: `once` (a date/time), `daily` or `weekly` (a time, weekly also a weekday), `interval` (every N days at a time). Optional `maxRuns` and `endsAt` stop a recurring job.
- Run now runs the job immediately without disturbing its schedule. Pause stops it; Delete removes it.
- Gated on the `cron` feature; the loop is idle unless cron is enabled, so customer boxes never run unattended agents.

## What you can ask

The job's prompt is whatever you want the agent to do at run time (reads, summaries, drafting, and writes that your Trust Rules permit). The agent does not create or schedule jobs itself.

## Limitations

- Owner-only for now; not exposed to customer boxes.
- Schedule presets only (once / daily / weekly / interval), no full cron expressions yet.
- A scheduled write with no matching Trust Rule queues rather than sending; check Approvals.
- Each run is a model call (token cost); use sensible frequency and `maxRuns` for campaigns.

## FAQ

<AccordionGroup>
  <Accordion title="Email X tomorrow morning.">
    New job, type Once, set the date and time, prompt = the email task. Add a Trust Rule for that send if you want it to go without approval.
  </Accordion>
  <Accordion title="One email a day for 30 days.">
    New job, Daily at a time, `maxRuns` 30; pair with a Trust Rule (optionally a 30-use rule) so each day's send auto-runs.
  </Accordion>
  <Accordion title="Why didn't my scheduled email send?">
    Likely no Trust Rule matched, so it queued in Approvals. Add a rule (Admin, then Trust) and it will send on the next run.
  </Accordion>
</AccordionGroup>
