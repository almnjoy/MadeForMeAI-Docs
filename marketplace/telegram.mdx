An approval-gated alert channel, plus a two-way chat path. The agent can queue a Telegram message
(an alert, reminder, or note), you approve it, and it sends to your chat through a bot. Outbound
alerts go to your **notify chat id**. To talk TO the agent from Telegram and get replies, add your
user id to the **Allowed user IDs** field on the same card: that opens the inbound gateway so the
agent answers you in chat. The card has three fields: a bot token, a notify chat id, and the
allowlist. Leave the notify chat id blank and it falls back to your first allowed id, so a
single-user setup can fill just the allowlist and skip the separate notify id.

## HOW TO SETUP

1. In Telegram, message **@BotFather**, send `/newbot`, follow the prompts, and copy the
   **bot token** it gives you (looks like `8875xxxxxx:AAH...`).
2. Open MindMap -> **Admin** (marketplace) -> the **Telegram** card.
3. Paste the **Bot token** into its write-only box, hit **Save**.
4. Get your chat id. Either message a chat-id bot (e.g. **@myidbot** or **@userinfobot**) and it
   replies with your id, or open **your** new bot and send it any message (e.g. `/start`). A bot
   cannot message you first, so messaging your bot is what registers your chat with it.
5. Back on the card, the **Notify chat id** (`TELEGRAM_CHAT_ID`, where approved alerts are sent)
   is found for you by the setup helper, which reads the chats your bot has seen. Paste the chat id
   into its box and **Save**. You can leave this blank if you fill the allowlist below: a blank
   notify id falls back to your first allowed id.
6. To chat with the agent from Telegram, paste your user id into the **Allowed user IDs** field
   (`TELEGRAM_ALLOWED_CHAT_IDS`) and **Save**. Use a comma-separated list for more than one person.
   Only ids in this list can talk to the agent and get replies.
7. Flip **Enable** on.

It applies on your next message, no restart.

## WHAT USER NEEDS TO GET

- **A bot token** from @BotFather (`/newbot`).
- **A notify chat id** (`TELEGRAM_CHAT_ID`), the destination for approved alerts. You do not have
  to hunt for it: message a chat-id bot (e.g. @myidbot or @userinfobot) or message your own bot
  once, then the setup helper (`GET /api/admin/telegram/chat-ids`) lists the chats your bot has
  seen with their ids. For a private chat it is a positive number; for a group it is a negative
  number. You can skip this field if you fill the allowlist, since a blank notify id falls back to
  your first allowed id.
- **Allowed user IDs** (`TELEGRAM_ALLOWED_CHAT_IDS`) for two-way chat. This is the same kind of id
  the helper shows you. Put one id to allow just yourself, or a comma-separated list for several
  people. Anyone not in this list cannot talk to the agent.

## HOW IT WORKS

- **Outbound notify:** the agent tool `notify_telegram` **queues** a message as an approval.
  Nothing sends yet. On approve, the `send_telegram` action calls the Telegram Bot API
  `sendMessage` with your token and your notify chat id. Messages render as HTML, up to 4000
  characters.
- **Inbound two-way:** when you add ids to **Allowed user IDs** (`TELEGRAM_ALLOWED_CHAT_IDS`), the
  inbound gateway turns on. Allowed users can message the bot, the agent runs your request, and it
  replies in the chat. Ids that are not on the list are ignored. Writes the agent proposes from a
  Telegram conversation are still approval-gated, same as everywhere else.
- The **notify chat id** falls back to your **first allowed id** when you leave the notify field
  blank. So a one-person setup can fill only the allowlist and still get outbound alerts.
- Token, notify chat id, and allowlist are read **live** via the marketplace key store (saved to
  your box, never shown back, never in the repo). Editing them applies on the next message, no
  restart.

## WHAT CAN BE CALLED VIA AI

- "send a telegram that the backup finished": queues an approval, sends on approve (write)
- "telegram me a reminder to call the vendor at 3": same, queued (write)
- any "notify / alert / ping my telegram" phrasing: queued (write)

From inside Telegram, an allowed user can also just chat the bot ("what's on my calendar today?",
"draft a reply to that email") and the agent answers there. Reads run immediately; any write it
proposes still waits for your approval. Nothing goes out on its own.

## LIMITATIONS

- **Two-way chat needs the allowlist.** With **Allowed user IDs** empty, the bot is outbound only
  and will not answer messages you send it. That silence is normal until you add your id.
- **Message the bot first.** Telegram will not give you a chat id until the bot has seen a
  message from you (bots cannot start a conversation).
- **One notify chat id** per box (the default outbound destination). You can target a group by
  using the group's chat id instead.
- **Allowlist is exact ids.** Only the ids you list can talk to the agent. There is no username or
  pattern matching, so a new person needs their id added by hand.
- **4000 character cap** on outbound sends, HTML parse mode.

## FAQ

**The bot does not reply when I message it. Is it broken?** No, the inbound gateway is off until
you add your id. Paste your user id into **Allowed user IDs** (`TELEGRAM_ALLOWED_CHAT_IDS`) on the
Telegram card and **Save**, then the agent answers you in chat. With that field empty the bot is
outbound only, so messaging it just registers your chat id.

**What is the notify chat id and where do I get it?** It is the outbound destination for approved
alerts. Message a chat-id bot (e.g. @myidbot or @userinfobot), or message your own bot once and the
setup helper lists it for you. Paste it into the Notify chat id box. If you fill the allowlist, you
can leave this blank and it uses your first allowed id.

**Can more than one person chat the bot?** Yes. Put each person's id in **Allowed user IDs** as a
comma-separated list. Every id on the list can talk to the agent; ids not on it are ignored.

**Can it send to a group?** Yes. Add the bot to the group, send a message there, and the helper
shows the group's (negative) chat id. Use that as the notify chat id.

**Is my token safe?** The key box is write-only. The token is saved to your box, never shown
back or returned to the browser, and never in the repo.

**My token leaked / I pasted it somewhere.** Message @BotFather, `/revoke`, get a fresh token,
and paste the new one in the same box. It applies live.
