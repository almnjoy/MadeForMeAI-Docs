client to your Composio server)


Let the agent use your Composio-connected apps (Gmail, Slack, GitHub, Notion, and 1000+ more).
MindMap connects to your Composio MCP endpoint and discovers the actions available there, then
exposes them to the agent. OAuth to each app is handled on Composio's side, so you do not hand the
app your Gmail or GitHub credentials. Read actions (list, search, fetch) run immediately; write
actions (send, create, update, delete) are approval-gated, so the agent queues them and they only
fire after you approve. This is the developer-grade connector lane; Zapier is the no-code lane.
Both live under Apps.

## HOW TO SETUP

1. Create a free **Composio** account (composio.dev). The free tier covers ~20k tool calls/month.
2. Create an **MCP server / tool router** and connect one or more apps (GitHub or Gmail are easy),
   enabling the actions you want. Composio runs the OAuth for each app.
3. Copy your **Composio MCP URL** (Streamable HTTP).
4. In MindMap, go to **Admin** (marketplace) and open the **Composio** card (it shows an MCP badge).
5. Paste the **Composio MCP URL** into its write-only box and **Save**.
6. If your endpoint sits behind a gateway that needs a consumer key, also paste the **Composio API
   key** into its box (it is sent as the `x-consumer-api-key` header). This is optional; most hosted
   URLs carry their own auth in the URL.
7. Flip **Enable** on.

It applies on your next message. New actions you enable on the Composio side appear within about a
minute (a short discovery cache).

## WHAT USER NEEDS TO GET

- **A Composio MCP URL** (Streamable HTTP) from your Composio MCP server / tool router. The URL is
  the credential for most setups.
- **(Optional) a Composio API key** if your endpoint is gateway-fronted and requires the
  `x-consumer-api-key` header. Connection still works on the URL alone when no key is needed.
- **The apps + actions you want, connected and enabled in the Composio dashboard.** MindMap shows
  whatever your Composio endpoint exposes; it does not configure apps for you.

## HOW IT WORKS

- MindMap is an **MCP client** to your Composio endpoint. It reads the URL (and optional key) via
  the marketplace key store (live, no restart) and handshakes to discover your available actions.
- Composio exposes tools in one of two shapes, and MindMap handles both:
  - **One tool per action** (e.g. `GITHUB_LIST_REPOSITORIES`, `GMAIL_SEND_EMAIL`).
  - **A tool-router meta-set** (e.g. `COMPOSIO_SEARCH_TOOLS` + `COMPOSIO_EXECUTE_TOOL`).
- **Reads run immediately, writes are approval-gated.** MindMap decides by the action name: names
  with GET / LIST / FETCH / SEARCH / READ / RETRIEVE / FIND / VIEW / COUNT / CHECK run live;
  everything else (and anything ambiguous) queues a proposal that only fires on your approval. The
  agent never sends or creates anything on its own.
- Results come back as a **structured tile** (a source badge, a success/error chip, key/value rows
  and tables) rather than a raw JSON blob, with a raw toggle and copy.
- App credentials (your Gmail, GitHub, etc.) stay on Composio's side. MindMap only holds the MCP
  URL and the optional consumer key.

## WHAT CAN BE CALLED VIA AI

Whatever you connected on the Composio side. For example, with GitHub + Gmail connected:

- "list my GitHub repos via Composio" (runs immediately, read)
- "create a GitHub issue titled X in <repo>" (queues an approval, runs on approve, write)
- "send an email via Composio to <addr>" (queued for approval, write)

The agent discovers your actions, runs reads, and queues writes for you.

## LIMITATIONS

- **Only what you enable in Composio** is available. MindMap mirrors your endpoint; it does not
  add integrations.
- **Writes are approval-gated** (by design). With the tool-router shape, a read run through the
  single executor is also gated (conservative but safe); a per-action "allow always" path can be
  added later. This ties into the planned grant-once / trust-rules approval direction.
- **About a minute of lag** when you enable a new action (the discovery cache refreshes on a TTL).
- **One MCP URL per box.** Treat the URL (and any key) as the credential; rotate on the Composio side.
- **Hosted free tier** is ~20k tool calls/month; self-hosting Composio is the path if you outgrow it.

## FAQ

**Composio or Zapier?** Both live under Apps and both are MCP connectors. Composio is the
developer-grade lane with OAuth handled per app and a large toolkit; Zapier is the no-code,
long-tail lane. Use either or both.

**Do I need the API key?** Only if your Composio endpoint is behind a gateway that requires the
`x-consumer-api-key` header. Otherwise the MCP URL alone connects.

**Can the agent act on its own?** No. Writes queue for your approval and only fire after you
approve. Reads run on their own.

**Where do my keys live?** The MCP URL and the optional API key are saved write-only to your pod,
never shown back, never in the repo. Rotate them on the Composio side if they leak.

**An action did not show up.** Make sure it is connected and enabled on your Composio MCP server,
then wait ~60s for the discovery cache to refresh.
