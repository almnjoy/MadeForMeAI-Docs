# Agent Browser

<Warning>
Agent Browser is in development and not yet available. The description below is what it will do once
it ships. The feature is not live in your workspace today.
</Warning>

A real, watchable Google Chrome browser running on an isolated VM that the agent can drive (real
Chrome stable via Patchright, not bundled Chromium, so it is far harder for sites to flag as a bot).
Reads (open a page, screenshot it, pull its text and links) run immediately. Interactions (click,
type) are approval-gated. The VM keeps a persistent profile so cookies and history accumulate.

## SETUP (PLANNED)
When it ships, you will connect it from Marketplace -> Agent Browser with two keys:
- `BROWSER_VM_URL` (the control API)
- `BROWSER_VM_TOKEN` (the control API bearer token)

## WHAT YOU WILL NEED
A reachable browser VM control API. This will be a managed cloud browser or your own VM. Read-only
endpoints needed: /navigate /extract /screenshot. Click/type will need /click and /type on the VM
(a later phase, not built yet).

## HOW IT WILL WORK
`browse` calls /navigate then /extract then /screenshot and renders an image tile (the screenshot)
plus a doc tile (the page text). The agent gets the title, status, first chunk of text, and the
links. `browser_action` queues an approval; on approve it calls the VM write endpoint.

## WHAT YOU WILL BE ABLE TO ASK
- "Open `<url>` and show me the page" -> browse
- "Read what's on `<url>`" -> browse
- "Click the Login button" / "type my email in the form" -> browser_action (approval-gated)

## LIMITATIONS
- Read-only today. Click/type need the VM Phase-2 endpoints.
- The VM browses untrusted sites; keep it isolated and snapshot-reverted.
- One shared browser context; not multi-user isolated (that is the managed-cloud path).

## FAQ
Q: Is it safe? A: Reads only by default. Any interaction is approval-gated. Never type a secret into
a page from a prompt; use a merchant API or enter it yourself.
Q: Can I watch it? A: Yes, a noVNC view behind your SSO.
