Reusable markdown playbooks the agent loads on demand. A skill is a named, editable markdown
document, a checklist the agent follows for a task: lead outreach, proposal/SOW, email drip, infra
triage, clean writing, brand voice. Add the ones you want; the agent pulls a skill's full text
only when a task matches, which keeps its context lean. You can let the agent pick a skill on its
own, or launch one yourself from the Chat App or the Canvas and pick a run mode. Writes inside a
skill stay approval-gated: the agent queues them and they fire only after you approve.

## HOW TO SETUP

Skills live in the marketplace **Skills** tab (no key needed). A small starter pack is seeded per
box. Tap **Add** on a skill to make it available to the agent, **Edit** to change its markdown, or
**New skill** to write your own.

## WHAT USER NEEDS TO GET

Nothing. Skills are stored per box in the database and edited in-app.

## HOW IT WORKS

- The agent's prompt carries a compact index of your **added** skills (slug + one-liner) so it
  knows what is available without loading everything.
- When a task matches, the agent calls `use_skill(slug)` to load the full playbook, then follows
  it.
- You can also **launch a skill yourself.** Open the skill launcher in the Chat App or on the
  Canvas, pick the skill, choose a run mode, and start it. You do not have to wait for the agent to
  decide a task matches.
- **Two run modes:**
  - **Normal** is the default. The agent runs the skill end to end. Reads run immediately. Any
    write the skill proposes still queues for your approval and fires only after you approve it.
  - **Plan** is a dry run. The agent reads what it needs and outlines the writes it would make,
    then stops. It writes nothing and queues nothing. Use it to see the plan before you commit, then
    run the skill in Normal mode when the plan looks right.
- Skills sit beside the Workflow Recorder (saved "plays"). A recorded play can become a skill
  later.

## WHAT CAN BE CALLED VIA AI

The agent calls `use_skill` itself. You just ask for the task ("draft a cold outreach to X", "write
me a proposal for Y"); if the matching skill is added, the agent loads and follows it.

You can also launch a skill by hand from the launcher in the Chat App or the Canvas:

- "launch the lead-outreach skill in Normal mode": the agent runs it; any writes queue for approval
- "launch the proposal-sow skill in Plan mode": the agent outlines the writes it would make and
  stops, writing nothing

## LIMITATIONS

- Only **added** (enabled) skills are visible to the agent; un-added ones sit on the shelf, and the
  launcher only lists added skills.
- Plan mode shows the writes the agent intends, not a guaranteed final result. The Normal run can
  still differ if the data changed in between.
- Skills are per box, not shared across boxes, for now.
- Each skill carries an optional agent-scope field (reserved for the future per-agent roster),
  unused today.

## FAQ

**How is a skill different from a note?** A note is content. A skill is an instruction the agent
follows.

**Where do they come from?** A seeded starter pack (stop-slop, brand-voice, lead-outreach,
proposal-sow, email-drip, infra-triage) plus any you create.

**What is the difference between Normal and Plan mode?** Normal runs the skill for real; reads
happen and any writes queue for your approval. Plan is a dry run that outlines the writes the agent
would make and stops, writing and queuing nothing. Run Plan first when you want to check the plan,
then run Normal to do it.

**Can a skill change or send anything without me?** No. Whether the agent picks a skill or you
launch it, every write queues for your approval and fires only after you approve. Reads run on
their own.
