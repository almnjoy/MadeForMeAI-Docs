Everything your agent can use lives in the marketplace, inside **Admin**. Each item is a toggle. A feature that is off still shows in the marketplace; it does nothing until you enable it and give it any key it asks for.

{/* MEDIA screenshot: the Admin marketplace with the four bucket tabs visible | caption: "The marketplace lives in Admin, split into four buckets." */}

## Two rules hold everywhere

<CardGroup cols={2}>
  <Card title="Reads run instantly" icon="bolt">
    Listing, searching, fetching, checking a balance: these happen live, the moment you ask, no approval needed.
  </Card>
  <Card title="Writes are approval-gated" icon="shield-check">
    Sending, creating, deleting, or moving anything queues for your approval first. Nothing fires on its own unless you set a narrow Trust Rule.
  </Card>
</CardGroup>

## The four buckets

The buckets stack on each other. Apps connect outside services and need a key. Tools use those Apps. Skills run Tools as steps. Secrets hold the keys.

<CardGroup cols={2}>
  <Card title="Apps" icon="plug">
    Connections to outside services: your CRM, Telegram, Money, Composio, and more. Most need a key or a URL. Each App has its own card with setup steps.
  </Card>
  <Card title="Tools" icon="wrench">
    The agent's built-in abilities, like web search, notes, and widgets. Most need no key. They draw on whatever Apps you have enabled.
  </Card>
  <Card title="Skills" icon="list-check">
    Reusable playbooks the agent runs on demand, like Deep Research or Lead Outreach. A Skill chains Tools into a sequence.
  </Card>
  <Card title="Secrets" icon="key">
    Your keys, stored write-only. You add and edit them here. The browser never shows a key back after you save it, and no key sits in our code.
  </Card>
</CardGroup>

## Enable a feature

<Steps>
  <Step title="Open the card">
    In Admin, find the marketplace and open the card for the feature you want.
  </Step>
  <Step title="Flip Enable on">
    Toggle the feature on.
  </Step>
  <Step title="Add its key or URL">
    If the card asks for a key or a URL, paste it. This saves into Secrets, write-only.
  </Step>
  <Step title="Save">
    The change applies on your next message, no restart.
  </Step>
</Steps>

<Note>
  A feature can show in the marketplace but be greyed as "needs key" until you add its Secret. That is expected. Add the key, Save, and it lights up.
</Note>
