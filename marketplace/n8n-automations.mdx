The "doing" layer. Where the other features read APIs directly, this bundle routes actions
through **your n8n** flows. Right now it covers: read and search your **M365 mailbox** (and reply
to or move a message, with approval), send a **templated email**, read your **calendar**
(M365 / cal.com), and **capture a lead** into Twenty. It's one toggle that says "my n8n is wired
up and powers these."

## HOW TO SETUP

1. Open MindMap, go to **Admin** (the marketplace).
2. Click the **n8n Automations** card.
3. Set your **n8n base URL** in the write-only box and Save.
4. Flip **Enable** on.

The catch that makes this different from the API features: the **flows have to exist in your
n8n**. MindMap calls webhooks; n8n does the work. The flow names/paths line up (`m365-email`,
`m365-calendar`, `send-templated-email`, `capture-lead`, etc.); reads use a fixed path, writes can
be repointed in your tenant config's `actionLayer.routes`.

## WHAT USER NEEDS TO GET

- An **n8n instance** reachable from MindMap (your `N8N_BASE_URL`).
- The **flows themselves**, built in n8n: a mailbox agent (read / search / open / reply / move /
  save across the agent + Dustin mailboxes via Microsoft Graph, the `M365-EMAIL-AGENT` flow,
  webhook `m365-email`), a templated-email send (renders the branded template + sends via Graph),
  a calendar read (M365 / cal.com), and a lead-capture (find-or-create in Twenty). These currently
  exist in Dustin's n8n; for a new customer they'd be tweaked or rebuilt for that customer's
  email/CRM/calendar.

## HOW IT WORKS

- For these three things, MindMap does **not** call email/calendar/CRM-write APIs itself. It
  POSTs to your n8n webhook (the action layer). n8n runs the flow and does the work.
- **Email is approval-gated.** The agent drafts an email and queues an approval card. Only when
  you approve does MindMap hit the n8n `send-templated-email` webhook, which renders the
  branded template (from the templates table) and sends via Graph. Nothing sends on its own.
- **Calendar** reads your real M365 / cal.com events through the n8n calendar flow.
- **Mailbox** reads and searches both mailboxes through the `m365-email` flow (Microsoft Graph),
  and can open a full message, list folders, and save an email for later; those run on the spot.
  **Reply and move are approval-gated:** the agent drafts the reply (or proposes the move), you
  approve, then MindMap hits the n8n flow which calls Graph. Inbound email text is treated as
  untrusted content: quoted to read, never followed as instructions.
- **Lead capture** posts a new lead to the n8n `capture-lead` flow, which find-or-creates it in
  Twenty (this is the input side; reading the pipeline is the Twenty CRM feature, direct API).
- This is why it's a separate bundle: it's flexible (n8n reaches 9,000+ apps) and per-customer
  (their flows, their email/CRM/calendar), where the direct-API features just need a key.

## WHAT CAN BE CALLED VIA AI

- "any new email?" / "look in my inbox for an email from George": reads / searches both mailboxes
- "open that one" / "what does it say": reads the full email body
- "reply: thanks, will do": queues a reply for approval, then n8n sends on approve
- "move it to the Customers folder": queues a move for approval
- "save this email": keeps it in your saved list to reference later
- "draft a <template> email to <person>": queues an approval, then n8n sends on approve
- "what's on my calendar": upcoming M365 / cal.com events
- "capture this as a lead": posts a new lead into Twenty via n8n

## LIMITATIONS

- **The bundle is the wiring, not the flows.** Enabling it points MindMap at your n8n; the
  flows must be built there. No n8n / no flows = these do nothing.
- **Per-customer.** Each customer's flows differ (their email sender, their CRM, their
  calendar), so this isn't a paste-a-key feature like the API ones, it's a build per customer.
- **Sending depends on the mailbox.** Reply and templated-email send go out as the mailbox. If
  M365 restricts that mailbox's outbound mail (common on a new tenant), Graph accepts the send but
  the message bounces. That's a mail-admin fix, not an MindMap one. Reading, searching, saving, and
  moving are unaffected.

## FAQ

**Does it send email without asking?** No. Email is approval-gated, the agent drafts, you
approve, then n8n sends.

**Why n8n instead of calling the APIs directly?** Flexibility and per-customer wiring. n8n can
reach almost anything, and each customer's email/CRM/calendar setup is different. The
read-only, every-customer-the-same things (pipeline, balances, metrics) are direct API.

**What if I don't run n8n?** Leave this off. The direct-API features (Twenty CRM read, Money,
Infra, Meraki) don't need it.

**Is the URL safe?** The box is write-only, saved to your pod, never shown back, never in the
repo.
